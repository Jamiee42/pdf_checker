# PDF Checker 需求文档 - v1.0

## 0. 项目背景
在翻译及排版过程中，常常需要统计 PDF 文件页面类型。不同页面类型所需的处理时长、资源、成本也存在差异：
| 页面类型 | 定义 | 处理特点 | 相对耗时 | 相对成本 |
|---|---|---|---|---|
|可编辑|包含可提取的文本层（Text Layer），支持直接选中、复制、搜索文本内容|可直接转换为可编辑格式（如通过 `另存为` 功能转为 Word、PPT 等），转换后文本内容准确性无需额外核验|短|低|
|不可编辑|无文本层或文本层不可提取，通常为扫描图像或嵌入式图片，文本内容仅以图像形式存在，无法直接选中或搜索|需经 OCR 识别以提取文本，识别结果需人工核验；识别失败或精度不足的部分需手动转录|长|高|
|空白|无文本层、扫描图像或嵌入式图片等可视内容|无需处理，根据项目需求保留或删除|可忽略|可忽略|
|无法识别|页面内容因加密或损坏等原因无法被识别|需要人工介入分析，依据具体情况确定解决方案|不确定|不确定|

人工统计往往费时费力，且易出现错漏、不准确的情况，本工具旨在将统计过程自动化，从而节省人工计算时间，提高项目管理效率。

## 1. 项目目标
开发一款桌面工具，用于判断 PDF 文件页面类型，并输出统计结果至 Excel，为报价、排期、人员统筹、成本控制提供数据支撑。

## 2. 用户故事
作为翻译公司人员，我希望能够快速获知 PDF 文件页面分布，以便：
- **客户服务**：制定报价方案
- **排版管理**：评估文件处理所需时长，进行项目规划
- **资源协调**：结合交期安排，控制成本预算，合理调配人员

## 3. 功能需求
| ID | 需求 | 优先级 |
|---|---|---|
|F01|用户可以选择并上传单个 PDF 文件|高|
|F02|遍历 PDF 所有页面|高|
|F03|判断 PDF 页面类型|高|
|F04|输出总页数，以及四种页面类型（可编辑、不可编辑、空白、无法识别）各自的页数和占比|高|
|F05|输出 F04 各类型页面对应的具体页码列表|高|
|F06|支持用户将 F04、F05 内容导出至 Excel 表格|高|
|F07|支持批量处理多个 PDF|中|
|F08|支持用户设定页面判断规则的阈值|中|
|F09|实时显示处理进度，提供处理 Log 供用户查阅|中|

## 5. 判断逻辑
- 设定判断阈值为 50
- 尝试提取页面文本
  - 第 1 层：文档加密 → 无法识别
  - 第 2 层：检测到文本层
     - 文本长度 ≥ 50 字符 → 可编辑
     - 文本长度 < 50 字符
         - 有可交互内容（表单、链接、注释） → 可编辑
         - 仅矢量图，无可交互内容 → 不可编辑
  - 第 3 层：未检测到文本层
     - 若有图像 → 不可编辑
     - 若无图像 → 空白页
*阈值说明：50 字符约为 1 行中文或 1 句英文，可避免将「仅包含页码、页眉等少量文本的页面」误判为可编辑。该值可根据实际使用场景调整。*

## 6. MVP 验收标准
- 运行脚本，输入 PDF 路径，能生成 Excel 文件
- Excel 包含：页面类型、文件名_对应页码
